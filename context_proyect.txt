## Gestión de Autenticación:
- Almacenamiento de token JWT en cookies para persistencia de sesión.
- Contexto de autenticación (AuthContext) para compartir el estado del usuario en toda la aplicación.
- Funciones para iniciar sesión, cerrar sesión y actualizar datos del usuario.
- Verificación automática de sesión al cargar la aplicación.
- Interceptores para incluir el token en las peticiones HTTP.
- Manejo de errores de autenticación (token expirado, inválido, etc.).
- Redirección a login cuando se intenta acceder a rutas protegidas sin autenticación.

### Implementación de Cookies:
- Se utiliza el objeto `document.cookie` para almacenar el token JWT.
- El token se guarda con atributos de seguridad (secure, samesite=strict).
- Se establece una fecha de expiración para la cookie (por defecto 7 días).
- Funciones implementadas en `tokenStorage.ts`:
  - `saveToken`: Guarda el token en una cookie
  - `getToken`: Obtiene el token desde la cookie
  - `removeToken`: Elimina la cookie (logout)
  - `hasToken`: Verifica si existe un token válido

### Flujo de Autenticación:
1. El usuario inicia sesión con username/email y contraseña
2. El backend valida las credenciales y devuelve un token JWT
3. El frontend guarda el token en una cookie
4. El token se envía automáticamente en cada petición HTTP
5. Si el token es inválido o expira, se redirige al usuario al login

## Consumo de API:
- Se utiliza Axios para realizar peticiones HTTP.
- Estructura de servicios separados por funcionalidad:
  - `authService`: Registro, login
  - `userService`: Operaciones con usuarios (perfil, seguir/dejar de seguir)
  - `postService`: Operaciones con publicaciones (crear, listar, actualizar, eliminar)
- Interceptores para añadir automáticamente el token de autenticación a las peticiones.
- Manejo centralizado de errores y respuestas.
- Tipado fuerte con TypeScript para las respuestas de la API.

### Endpoints Principales:
- Autenticación: `/api/auth/signup`, `/api/auth/login`
- Usuarios: `/api/users/profile`, `/api/users/{username}`, `/api/users/{username}/follow`
- Publicaciones: `/api/post/posts`, `/api/post/feed`, `/api/post/posts/{postId}`

## Páginas Implementadas:
